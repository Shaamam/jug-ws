# Code Explainer

When asked to explain code:

1. **One-Line Summary**: Start with a single sentence describing what the code does
2. **Section Breakdown**: Walk through each significant section or line
3. **Concepts**: Identify and explain any design patterns, algorithms, or language features used
4. **Potential Issues**: Highlight any bugs, edge cases, or improvement opportunities
5. **Analogy**: If the concept is complex, provide a real-world analogy
6. **Simplified Example**: If helpful, show a minimal version of the same concept

Use plain English. Assume the reader knows basic programming but may not know the specific language or framework.
